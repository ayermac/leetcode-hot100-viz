# Research Summary

**Project:** LeetCode Hot 100 可视化学习平台
**Date:** 2026-04-30

---

## Overview

研究完成。本文档总结了算法可视化学习平台的技术栈、功能、架构和潜在陷阱。

---

## Key Findings

### 推荐技术栈

| 层级 | 技术 | 版本 | 理由 |
|------|------|------|------|
| **动画** | Framer Motion | 12.x | 声明式 API，布局动画，React 原生集成 |
| **图/树可视化** | React Flow | 12.x | 节点图专用，内置交互，自定义节点支持 |
| **UI 组件** | shadcn/ui | Latest | 可访问性，Tailwind 原生，代码可控 |
| **样式** | Tailwind CSS | 4.x | 零配置，5x 构建速度，现代 CSS |
| **Markdown** | next-mdx-remote | 6.x | 灵活加载，RSC 支持，安全默认 |
| **代码高亮** | Shiki | 4.x | VS Code 级准确度，零客户端 JS |
| **部署** | Vercel | - | Next.js 原生，免费层，全球 CDN |

### 核心功能优先级

**MVP 必需:**
1. 步进执行控制（播放/暂停/步进/重置）
2. 速度控制（0.25x - 2x）
3. 代码同步高亮
4. 自定义输入数据
5. 数组和链表可视化器

**后续增强:**
- 交互式测验
- 并排比较模式
- 递归树/调用栈可视化
- 进度追踪

### 架构要点

```
用户输入 → 算法执行器 → 状态快照 → 动画控制器 → 可视化引擎
     ↑                                           ↓
     └──────────── 控制反馈 ────────────────────────┘
```

**构建顺序:**
1. 内容解析管道（markdown → JSON）
2. 页面结构 & 路由
3. 基础问题展示
4. 动画状态机
5. 数组可视化器
6. 链表可视化器

### 关键陷阱警告

| 陷阱 | 预防措施 |
|------|----------|
| 动画太快 | 默认 0.5x-1x，提供细粒度控制 |
| 状态丢失上下文 | 高亮变化，显示操作标签 |
| 链表自动布局 | 禁止自动重排，保持空间位置 |
| 静态导出错误 | 从一开始使用 `output: 'export'` |
| Markdown 不一致 | 创建验证脚本，严格 schema |
