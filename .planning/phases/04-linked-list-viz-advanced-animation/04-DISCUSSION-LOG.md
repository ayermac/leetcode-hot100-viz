## 链表布局样式

### 链表节点排列方式

| Option | Description | Selected |
|--------|-------------|----------|
| 水平布局 | 节点从左到右排列，箭头连接。适合较短的链表，与数组可视化风格一致。 | ✓ |
| 垂直布局 | 节点从上到下排列，适合长链表。但可能与数组可视化风格不一致。 | |
| 自适应布局 | 根据链表长度自适应切换。增加复杂度，但更灵活。 | |

**User's choice:** 水平布局（推荐）

### 环形链表展示方式

| Option | Description | Selected |
|--------|-------------|----------|
| 虚线回环箭头 | 用虚线箭头从尾节点指向环入口节点，直观展示环的结构。 | ✓ |
| 快慢指针相遇动画 | 不特殊展示环，仅通过快慢指针相遇来暗示存在环。 | |
| 循环排列 | 节点排列成圆形，环内节点围成一圈。直观但实现复杂，可能打乱水平布局。 | |

**User's choice:** 虚线回环箭头（推荐）

### 节点渲染样式

| Option | Description | Selected |
|--------|-------------|----------|
| 值+指针分栏 | 左半部分显示节点值，右半部分显示 next 指针（箭头）。类似教科书图示。 | ✓ |
| 仅显示值 | 只显示值，箭头从节点边缘出发指向下一个节点。更简洁。 | |

**User's choice:** 值+指针分栏（推荐）

### 指针标注显示

| Option | Description | Selected |
|--------|-------------|----------|
| 节点上方标签 | 类似数组的 PointerArrow，在节点上方显示 slow、fast、prev、curr 等指针名称。 | ✓ |
| 节点边框颜色 | 直接在节点边框上添加彩色标记，如 slow 用绿色边框，fast 用蓝色边框。 | |

**User's choice:** 节点上方标签（推荐）

## 代码同步高亮

### 代码高亮实现方式

| Option | Description | Selected |
|--------|-------------|----------|
| 行背景高亮+自动滚动 | 当前执行行的背景色高亮，随动画步骤自动滚动到对应行。类似调试器体验。 | ✓ |
| 仅行背景高亮 | 仅高亮当前行，不自动滚动，用户手动滚动查看。 | |
| 行号标记 | 行号旁边显示当前步骤标记，如 ▶ 图标。 | |

**User's choice:** 行背景高亮+自动滚动（推荐）

### 代码与动画布局关系

| Option | Description | Selected |
|--------|-------------|----------|
| 代码在上，动画在下 | 上方显示代码，下方显示可视化动画，类似 IDE 调试布局。宽屏体验好。 | ✓ |
| 代码在左，动画在右 | 左侧显示代码，右侧显示动画。窄屏体验好，但代码行宽受限。 | |
| Tab 切换 | 用户可切换 Tab 查看代码或动画。节省空间但无法同时看到代码和动画。 | |

**User's choice:** 代码在上，动画在下（推荐）

### 步骤描述文字位置

| Option | Description | Selected |
|--------|-------------|----------|
| 动画下方显示 | 在动画下方显示当前步骤的文字描述，如"快指针移动到节点 2"。 | ✓ |
| 代码下方显示 | 在代码下方显示当前步骤描述。 | |
| 中间分隔区显示 | 在代码和动画之间的分隔区域显示。 | |

**User's choice:** 动画下方显示（推荐）

## 自定义输入交互

### 链表输入 UI 形式

| Option | Description | Selected |
|--------|-------------|----------|
| 文本框逗号分隔 | 用户输入如 1,2,3,4,5，解析为链表。简单直观，与 LeetCode 输入格式一致。 | ✓ |
| 可视化节点编辑器 | 点击添加节点，填写值，可拖拽排序。更直观但操作繁琐。 | |
| 两种方式都支持 | 同时提供文本框和可视化编辑器。灵活但 UI 复杂。 | |

**User's choice:** 文本框逗号分隔（推荐）

### 带环链表输入方式

| Option | Description | Selected |
|--------|-------------|----------|
| 最后一个值表示环入口 | 输入如 1,2,3,4,2（最后一个数字是环入口节点值）。简洁。 | ✓ |
| 分号分隔环入口 | 输入如 1,2,3,4;2（分号后是环入口位置）。更明确但格式复杂。 | |
| 仅预设用例支持环 | 普通输入不支持环，选择预设用例"环形链表"来展示环。 | |

**User's choice:** 最后一个值表示环入口（推荐）

### 预设测试用例数量

| Option | Description | Selected |
|--------|-------------|----------|
| 3-5 个预设 | 提供 3-5 个典型用例，如"基础用例""边界情况""环形链表"。 | ✓ |
| 10+ 个预设 | 每个算法提供 10+ 个预设用例，覆盖各种边界情况。 | |
| 无预设 | 不提供预设，用户完全自定义输入。 | |

**User's choice:** 3-5 个预设（推荐）

### 输入验证错误展示

| Option | Description | Selected |
|--------|-------------|----------|
| 输入框下方提示 | 输入无效时，在输入框下方显示红色错误提示，禁用运行按钮。 | ✓ |
| Toast 提示 | 弹出 Toast 提示错误信息。不阻挡用户操作。 | |

**User's choice:** 输入框下方提示（推荐）

## 首批链表算法

### 算法选择优先级

| Option | Description | Selected |
|--------|-------------|----------|
| 基础指针操作 | 反转链表、环形链表检测、合并两个有序链表、删除倒数第N个节点。基础且代表性。 | ✓ |
| 多样化技巧 | 反转链表、环形链表检测、回文链表、相交链表。更多样化但难度适中。 | |
| 最小集合 | 只实现反转链表和环形链表检测，作为 MVP 验证架构。 | |

**User's choice:** 基础指针操作（推荐）
