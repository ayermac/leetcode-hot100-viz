# 数组中的第K个最大元素：用堆快速找出排名！

大家好，我是小编。今天我们来攻克一道经典的堆应用题目 - LeetCode 215「数组中的第K个最大元素」。这个问题在面试中非常常见，也是理解堆这个数据结构的绝佳案例。

## 📚 生活中的场景

想象你是一个老师，手里有一叠试卷成绩。现在你想知道：第K名考了多少分？直接全部排序当然可以，但如果班级有1000人，你只需要知道第10名，排序就显得有点大材小用了。这就是我们今天要解决的问题！

## 💡 问题详解

**题目要求**：
给定整数数组 nums 和整数 k，请返回数组中第 k 个最大的元素。

**举个简单的例子**：
```go
输入：nums = [3,2,1,5,6,4], k = 2
输出：5
解释：排序后是 [1,2,3,4,5,6]，第2大的元素是5
```

## 🤔 解题思路发展

让我们一步步发展解题思路：

1. **暴力解法**
   直接排序，然后返回第k大的元素。时间复杂度O(nlogn)。

2. **优化方向**
   我们只需要第k大的元素，不需要对所有元素排序。堆正好可以帮我们维护前k个元素！

3. **小顶堆的应用**
   维护一个大小为k的小顶堆，堆顶就是第k大的元素。因为小顶堆的堆顶是最小元素，而堆中保持k个最大的元素，所以堆顶就是第k大！

## 🚀 代码实现

### 方法一：小顶堆（推荐）

```go
// 小顶堆类型
type MinHeap []int

func (h MinHeap) Len() int           { return len(h) }
func (h MinHeap) Less(i, j int) bool { return h[i] < h[j] }
func (h MinHeap) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }

func (h *MinHeap) Push(x interface{}) {
    *h = append(*h, x.(int))
}

func (h *MinHeap) Pop() interface{} {
    old := *h
    n := len(old)
    x := old[n-1]
    *h = old[0 : n-1]
    return x
}

func findKthLargest(nums []int, k int) int {
    // 创建小顶堆
    h := &MinHeap{}
    
    for _, num := range nums {
        heap.Push(h, num)
        // 保持堆的大小为k
        if h.Len() > k {
            heap.Pop(h)
        }
    }
    
    // 堆顶就是第k大的元素
    return (*h)[0]
}
```

### 方法二：大顶堆

```go
// 大顶堆类型
type MaxHeap []int

func (h MaxHeap) Len() int           { return len(h) }
func (h MaxHeap) Less(i, j int) bool { return h[i] > h[j] } // 大顶堆：大的在前
func (h MaxHeap) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }

func (h *MaxHeap) Push(x interface{}) {
    *h = append(*h, x.(int))
}

func (h *MaxHeap) Pop() interface{} {
    old := *h
    n := len(old)
    x := old[n-1]
    *h = old[0 : n-1]
    return x
}

func findKthLargest(nums []int, k int) int {
    // 创建大顶堆
    h := &MaxHeap{}
    
    // 将所有元素加入堆
    for _, num := range nums {
        heap.Push(h, num)
    }
    
    // 弹出前k-1个元素
    for i := 0; i < k-1; i++ {
        heap.Pop(h)
    }
    
    // 堆顶就是第k大的元素
    return (*h)[0]
}
```

### 方法三：快速选择（最优时间复杂度）

```go
func findKthLargest(nums []int, k int) int {
    // 第k大就是第n-k小
    target := len(nums) - k
    left, right := 0, len(nums)-1
    
    for left < right {
        pivot := partition(nums, left, right)
        if pivot == target {
            break
        } else if pivot < target {
            left = pivot + 1
        } else {
            right = pivot - 1
        }
    }
    
    return nums[target]
}

func partition(nums []int, left, right int) int {
    pivot := nums[right]
    i := left
    for j := left; j < right; j++ {
        if nums[j] < pivot {
            nums[i], nums[j] = nums[j], nums[i]
            i++
        }
    }
    nums[i], nums[right] = nums[right], nums[i]
    return i
}
```

## 📝 方法比较

1. **小顶堆法**
   - 时间复杂度：O(nlogk)
   - 空间复杂度：O(k)
   - 优点：适合处理大数据流，只需要维护k个元素

2. **大顶堆法**
   - 时间复杂度：O(nlogn)
   - 空间复杂度：O(n)
   - 优点：思路直观
   - 缺点：需要存储所有元素

3. **快速选择**
   - 时间复杂度：O(n) 平均，O(n²) 最坏
   - 空间复杂度：O(1)
   - 优点：平均最优时间复杂度
   - 缺点：最坏情况会退化

## 🎯 易错点剖析

1. **堆的大小控制**
   - 小顶堆保持k个元素，超出时弹出堆顶
   - 大顶堆需要先全部加入，再弹出k-1个

2. **Go中堆的使用**
   - 必须实现 heap.Interface 的5个方法
   - Push/Pop 是指针方法，Len/Less/Swap 是值方法
   - 调用时使用 heap.Push 和 heap.Pop

3. **索引理解**
   - 第k大 = 排序后倒数第k个
   - 对于快速选择：第k大 = 第n-k小（0索引）

## 🌟 面试技巧

1. **思路解释**
   - 先说明为什么可以用堆解决
   - 解释小顶堆和大顶堆的区别

2. **复杂度分析**
   - 小顶堆：O(nlogk) 时间，O(k) 空间
   - 快速选择：O(n) 平均时间，O(1) 空间

3. **优化建议**
   - 数据量大且k小时，用小顶堆
   - 追求最优时间复杂度，用快速选择
